1.什么是哈希表？什么是哈希集合？
  ```text
  1> 哈希表 = 用哈希函数实现的“键值对（key-value）存储结构”
  哈希表就是“用 key 快速找到 value”
  2> 哈希集合 = 只存“key”的哈希表（没有 value）
  哈希集合就是“快速判断某个元素是否存在”
  ```

2.哈希表的成员函数pairs.count(ch)进行什么操作？
  ```text
  unordered_map<char, char> pairs = {
    {')', '('},
    {']', '['},
    {'}', '{'}
  };
  这里的结构是：
  key   → value
  ')'   → '('
  ']'   → '['
  '}'   → '{'
  pairs.count(ch) 的含义是：判断 ch 是否是 map 里的 key
  ```

3.可能可以的思考路径
  ```text
  1> 这道题的本质是什么？
  括号是否“成对 + 顺序正确 + 嵌套正确”
  2> 为什么能想到“栈”？
  观察：后出现的左括号，必须先匹配
  后进先出（LIFO）
  自然得到：用栈存“未匹配的左括号”
  3> 只用栈够不够？（哈希表登场的原因）
  算法雏形：遇到左括号 → 入栈；遇到右括号 → 和栈顶匹配
  关键问题来了：如何判断 “右括号应该匹配哪个左括号？”
  ```

4.算法
  ```text
  Step 1：长度必须是偶数
  if (n % 2 == 1) return false;
  Step 2：用栈处理左括号
  遇到左括号 → 存起来（等待匹配）
  Step 3：遇到右括号
  必须匹配“最近的左括号”
  Step 4：匹配规则
  栈顶 == 这个右括号对应的左括号
  这里用哈希表实现
  Step 5：全部处理完
  return stk.empty();
  必须全部匹配掉
  ```

5.为什么用哈希表和栈？
  ```text
  1> 为什么用哈希表：匹配关系问题（映射）
  “这是一个映射问题”
  看到：
  ')' 要匹配 '('
  ']' 要匹配 '['
  应该意识到：这是一个 key → value 的映射
  2> 为什么用栈：用来解决：顺序 + 嵌套问题
  ```
